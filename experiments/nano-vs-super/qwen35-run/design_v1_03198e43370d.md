# Create implementation design for: Authentication patterns for identity services (prd_id: c69771de002c) (project_id: PRJ-963bc59e) — Implementation Design

This design implements a robust, NIST SP 800-63-4 compliant authentication system for the IMDB Lite context, utilizing TypeScript, Node.js, Express, MongoDB (Mongoose), and Redis. It addresses the identified threats (THR-001 Spoofing, THR-002 Tampering) through short-lived JWTs, Redis-backed Token Revocation Lists (TRL), and strict input validation.

## 2. API Endpoint Specifications

All endpoints follow RESTful conventions. Authentication is required for all endpoints except `/auth/register` and `/auth/login`.

### Authentication Endpoints

| Method | Endpoint | Description | Auth Required | RBAC Claim | Response Headers |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `POST` | `/api/v1/auth/register` | Register new user | No | `n/a` | `Set-Cookie: refresh_token` |
| `POST` | `/api/v1/auth/login` | User login | No | `n/a` | `Set-Cookie: refresh_token`, `X-Access-Token: jwt` |
| `POST` | `/api/v1/auth/refresh` | Issue new Access Token | Yes (Refresh) | `n/a` | `Set-Cookie: refresh_token` |
| `POST` | `/api/v1/auth/logout` | Revoke tokens | Yes (Access) | `n/a` | `Set-Cookie: refresh_token=; expires=0` |
| `GET` | `/api/v1/auth/me` | Get current user profile | Yes (Access) | `viewer`, `reviewer`, `admin` | `Content-Type: application/json` |

### Protected Data Endpoints (Movies)

| Method | Endpoint | Description | Auth Required | RBAC Claim |
| :--- | :--- | :--- | :--- | :--- |
| `GET` | `/api/v1/movies` | List all movies | Yes | `viewer` (default public) |
| `GET` | `/api/v1/movies/:id` | Get single movie | Yes | `viewer` (default public) |
| `POST` | `/api/v1/movies` | Create movie | Yes | `admin` |
| `PUT` | `/api/v1/movies/:id` | Update movie | Yes | `reviewer` or `admin` |
| `DELETE` | `/api/v1/movies/:id` | Delete movie | Yes | `admin` |

### Redis Token Revocation Structure
We use Redis for the Token Revocation List (TRL) to meet the 100ms lookup requirement (FR-04).

*   **Key**: `revoked:<access_token_jti>` (e.g., `revoked:a1b2c3-d4e5-f6`)
*   **Value**: Empty string or timestamp
*   **TTL**: Matches Access Token expiration (15 minutes).

## 4. Authentication Middleware Implementation

### JWT Logic & Validation
Uses `RS256` (preferred for production) or `HS256` with strong secrets. Ensures strict claim validation.

```typescript
// src/shared/utils/token.util.ts
import jwt from 'jsonwebtoken';
import { config } from '../../config';

interface TokenOptions {
  expiresIn: string; // e.g., '15m'
}

export const generateTokens = (userId: string, role: string) => {
  const payload = {
    userId,
    role,
    // Generate a unique JTI (JSON Token ID) for revocation tracking
    jti: crypto.randomUUID(), 
  };

  const accessOptions: TokenOptions = { expiresIn: '15m' };
  const refreshOptions: TokenOptions = { expiresIn: '7d' };

  const accessToken = jwt.sign(payload, config.JWT_SECRET_ACCESS, accessOptions);
  const refreshToken = jwt.sign(payload, config.JWT_SECRET_REFRESH, refreshOptions);

  return { accessToken, refreshToken, jti: payload.jti };
};

export const verifyAccessToken = (token: string) => {
  try {
    return jwt.verify(token, config.JWT_SECRET_ACCESS) as jwt.JwtPayload;
  } catch (err) {
    throw new jwt.JsonWebTokenError('Token verification failed');
  }
};

export const verifyRefreshToken = (token: string) => {
  try {
    return jwt.verify(token, config.JWT_SECRET_REFRESH) as jwt.JwtPayload;
  } catch (err) {
    throw new jwt.JsonWebTokenError('Refresh token verification failed');
  }
};
```

### RBAC Middleware
Implements "Deny-by-Default" (FR-03) and validates role claims.

```typescript
// src/shared/middleware/auth.middleware.ts
import { Request, Response, NextFunction } from 'express';
import { config } from '../../config';
import { verifyAccessToken, generateTokens } from '../utils/token.util';
import { redisClient } from '../utils/redis.util';
import { v4 as uuidv4 } from 'uuid';

export const authenticate = async (req: Request, res: Response, next: NextFunction) => {
  const authHeader = req.headers.authorization;

  if (!authHeader || !authHeader.startsWith('Bearer ')) {
    return res.status(401).json({ error: 'Missing or invalid Authorization header' });
  }

  const token = authHeader.split(' ')[1];
  
  try {
    const decoded = verifyAccessToken(token);
    
    // Check Revocation List (THR-001 Mitigation)
    // Key format: revoked:<jti>
    if (decoded.jti) {
      const isRevoked = await redisClient.get(`revoked:${decoded.jti}`);
      if (isRevoked) {
        return res.status(401).json({ error: 'Token has been revoked' });
      }
    }

    // Attach user info to request object
    req.user = decoded as any;
    next();
  } catch (err) {
    return res.status(401).json({ error: 'Invalid or expired token' });
  }
};

export const authorize = (...allowedRoles: string[]) => {
  return (req: Request, res: Response, next: NextFunction) => {
    if (!req.user || !allowedRoles.includes(req.user.role)) {
      return res.status(403).json({ error: 'Insufficient permissions' });
    }
    next();
  };
};
```

### Refresh Token Logic
Implements Refresh Token Rotation (FR-02).

```typescript
// src/modules/auth/auth.controller.ts
import { Router } from 'express';
import { authService } from './auth.service';
import { authenticate } from '../../shared/middleware/auth.middleware';
import { Request, Response } from 'express';
import { redisClient } from '../../shared/utils/redis.util';

const router = Router();

// Refresh Endpoint
router.post('/refresh', authenticate, async (req: Request, res: Response) => {
  try {
    // Invalidate old refresh token (rotation)
    // In a real scenario, we would check the refresh token stored in DB/Redis
    // Here we assume the refresh token is passed in a cookie or header
    
    const newTokens = await authService.generateTokens(req.user.userId, req.user.role);
    
    // Store new refresh token in Redis (optional, for rotation logic)
    // const oldRefreshToken = req.cookies.refresh_token; 
    // await redisClient.set(`refresh:${oldRefreshToken}`, 'invalid', 'NX'); 

    // Set new access token
    res.setHeader('X-Access-Token', newTokens.accessToken);
    
    // Set new refresh token in HttpOnly cookie
    res.cookie('refresh_token', newTokens.refreshToken, {
      httpOnly: true,
      secure: config.NODE_ENV === 'production',
      sameSite: 'strict',
      maxAge: 7 * 24 * 60 * 60 * 1000, // 7 days
    });

    res.json({ 
      message: 'Token refreshed', 
      user: { id: req.user.userId, role: req.user.role } 
    });
  } catch (error) {
    res.status(500).json({ error: 'Token refresh failed' });
  }
});

export { router as authRouter };
```

### Input Validation (NoSQL Injection Prevention)
Use `Joi` or `Zod` to validate inputs strictly before they reach the controller.

```typescript
// src/shared/middleware/validation.ts
import { Request, Response, NextFunction } from 'express';
import { z } from 'zod';

export const validateRequest = (schema: z.ZodSchema) => {
  return (req: Request, res: Response, next: NextFunction) => {
    const result = schema.safeParse(req.body);
    
    if (!result.success) {
      return res.status(400).json({ 
        error: 'Validation failed', 
        details: result.error.errors 
      });
    }
    
    // Cast valid input to request body
    req.body = result.data as any;
    next();
  };
};
```

### Global Error Handler
```typescript
// src/shared/middleware/error.middleware.ts
import { Request, Response, NextFunction } from 'express';
import { AppError } from '../utils/errors';
import { config } from '../../config';

export const errorHandler = (
  err: Error | AppError,
  req: Request,
  res: Response,
  next: NextFunction
) => {
  let statusCode = 500;
  let message = 'Internal Server Error';

  if (err instanceof AppError) {
    statusCode = err.statusCode;
    message = err.message;
  }

  // In development, include stack trace
  const errorResponse = {
    success: false,
    statusCode,
    message,
    ...(config.NODE_ENV === 'development' && { stack: err.stack }),
  };

  // Log error (masked)
  console.error(`[${statusCode}] ${message}`);

  res.status(statusCode).json(errorResponse);
};
```
