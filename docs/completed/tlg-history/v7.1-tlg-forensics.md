# TLG Forensics Review — v7.1 adapter + temporal stack

**Date:** 2026-04-23
**Adapter under test:** `adapters/checkpoints/software_dev/v7.1/`
**Stack flags:** `NCMS_TEMPORAL_ENABLED=true`, `NCMS_SLM_ENABLED=true`

## TL;DR

TLG is **wired correctly** but **starved of data** at two points. The dispatcher, per-shape walkers, and grammar composition are structurally sound. Two upstream producers are broken:

1. **`shape_intent` head** is template-overfit — 100% on training, 25.6% on held-out natural queries. Real user queries all classify as `none` with high confidence, bypassing the entire TLG layer.
2. **L2 state-node creation** stores the raw sentence as `state_value` instead of the canonical entity (e.g. "postgresql"). Reconciliation can't find supersession partners and emits zero `supersedes` / `derived_from` / `refines` edges.

The result: even after ingesting a narrative with 3 explicit supersessions (PostgreSQL → CockroachDB → YugabyteDB), the zone graph has zero walkable edges and 7/10 dispatchers ABSTAIN when fed the right `shape_intent` directly.

## Forensics A — shape_intent head accuracy

| split | N | accuracy |
|-------|--:|---------:|
| training gold | 181 | **1.000** |
| held-out (natural queries) | 39 | **0.256** |

All 181 training queries share **12 template prefixes**, one per shape:

```
17× "What decision was adopted in: <ADR>?"               → current_state
17× "What problem existed before the decision was made:" → before_named
17× "What background or problem motivated the decision:" → origin
17× "What closing summary does the decision record:"     → ordinal_last
...
```

The head learned the scaffold, not the shape. Held-out behaviour:

```
gold=current_state  pred=none  conf=0.94  "Which framework are we currently running in production?"
gold=before_named   pred=none  conf=0.97  "What did we use before we switched to Postgres?"
gold=before_named   pred=none  conf=1.00  "Which ORM predated Prisma in the codebase?"
gold=retirement     pred=none  conf=0.99  "Which technologies did we deprecate last quarter?"
gold=origin         pred=none  conf=1.00  "How did we end up using this database?"
```

**20 of 29 wrong predictions are high-confidence (>0.7) `none`** — exactly the worst failure mode for a router (confidently sends real traffic to the bypass path).

**Verdict on dispatcher:** the shape_intent → dispatcher-intent mapping is correct (10/10 routed to the right walker), L3 subject extraction works (6/10 extracted a correct subject entity), and `_SLM_SHAPE_TO_DISPATCH_INTENT` at `dispatch.py:605` passes through cleanly. The dispatcher is not the problem.

**Why 7/10 ABSTAIN:** the walkers traverse the zone graph (supersedes/refines/derived_from edges on L2 entity_state nodes). That graph is empty — see Forensics C.

## Forensics C — state reconciliation audit

After ingesting 5 memories with explicit state transitions:

```json
{
  "l1_atomic_count": 5,             // ✓ expected 5
  "l2_entity_state_count": 2,       // ✗ expected ~5 (one per transition)
  "supersedes_edges": 0,            // ✗ expected 2 (PG→CRDB, CRDB→Yugabyte)
  "derived_from_edges": 0,          // ✗ expected 5 (L1→L2 per memory)
  "refines_edges": 0,               // ✗ expected 1 (Patroni refines PG)
  "conflicts_edges": 0,             // ✓ expected 0
  "l2_current": 2,
  "l2_retired": 0                   // ✗ CRDB should be retired
}
```

The 2 L2 nodes that **did** get created expose the root cause — `state_value` is the raw sentence:

```json
{
  "entity_id": "auth-service",
  "state_key": "state",
  "state_value": "The auth-service now uses YugabyteDB in all three regions as of the latest rollout."
},
{
  "entity_id": "cockroachdb",
  "state_key": "state",
  "state_value": "We have deprecated CockroachDB in favor of YugabyteDB for stronger Postgres compatibility."
}
```

Reconciliation matches state nodes by `(entity_id, state_key, state_value)`. With the value field being an entire sentence, no two memories ever match, so no `supersedes` / `refines` edges fire. The `is_current=True` flag on the retired CRDB node confirms reconciliation never ran.

**Diagnosis:** the L2 builder is storing memory content, not extracted entity values. The SLM is producing `slots["database"] = "yugabytedb"` correctly (v7.1 role head has 0.79 F1), but that canonical slot value isn't flowing into the L2 node's `state_value` field.

## Forensics D — end-to-end auto-classify

All 4 natural queries return `Confidence.NONE` with empty intents — direct consequence of Forensics A. The shape_intent head predicts `none`, `retrieve_lg()` short-circuits at `dispatch.py:729` and returns an empty LGTrace.

| component | severity | impact |
|-----------|----------|--------|
| shape_intent training corpus is 100% template-wrapped | **HIGH** | Blocks end-to-end entirely — every natural query routes to `none` with high confidence |
| L2 `state_value` = raw sentence instead of canonical slot value | **HIGH** | Zero supersedes/refines edges; 7/10 walkers ABSTAIN; `is_current` flag never flips |

## Prescription

### Fix 1 — Retrain shape_intent head on natural queries
**Effort:** 1–2 hours.

Generate ~300–500 natural query phrasings per shape via Spark Nemotron with explicit "phrase this like a real user" instruction. Keep the 181 template rows (they're still useful signal), add the natural ones alongside. Retrain v7.2 with identical hyperparams — the 5 other heads are already strong, only this one needs refresh.

Expected held-out accuracy lift: 0.26 → 0.80+ (similar to the v7.1 slot-F1 curve once we diversified training).

### Fix 2 — Wire the role head's primary slot into L2 `state_value`
**Effort:** 0.5–1 hour, one file.

The v7.1 role head already emits canonical slot values (`{"database": "yugabytedb"}`). Somewhere in the ingestion pipeline (probably `ingestion/pipeline.py::_create_l2_node` or `memory_service.store_memory`), the code writes `state_value=<raw text>` instead of `state_value=<primary slot canonical>`. A 5-line change: pick the primary-role span's canonical as the L2 value.

Once Fix 2 lands, existing v7.1 adapter generates proper zone graphs → most of Forensics B's ABSTAINs flip to answers without retraining.

### Fix 3 — Ensure every state-bearing memory produces an L2 node
**Effort:** Investigate first.

Only 2/5 ingested memories produced L2 nodes. Need to trace: is the state-declaration detection (`state_change` head → `declaration`) firing for all 5? If so, why isn't L2 creation following? Suspect a guard in ingestion that skips L2 when `state_value` can't be determined — with Fix 2 this may self-resolve.

### Priority order

1. **Fix 2** first (canonical state_value). Quickest, unblocks Forensics B without retraining.
2. **Fix 3** investigation (L2 coverage). Likely fixed-by-2, verify.
3. **Fix 1** (shape_intent retrain). Biggest effort, unblocks Forensics D.

After all three, re-run this forensics script. Expected results:
- A: held-out ≥ 0.80
- C: `supersedes_edges ≥ 2`, `l2_retired ≥ 1`
- B: at least 7/10 walkers return HIGH/MEDIUM
- D: 3/4 natural queries route + return answers
