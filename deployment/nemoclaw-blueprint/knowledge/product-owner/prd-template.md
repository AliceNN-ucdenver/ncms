# PRD Template — Product Requirements Document

Use this template when creating a PRD. All sections are required.

## 1. Problem Statement
What problem are we solving? Who is affected? What is the impact of not solving it?

## 2. Goals
- **Primary Goal:** What must this achieve?
- **Secondary Goals:** Nice-to-have outcomes
- **Non-Goals:** What is explicitly out of scope?

## 3. User Stories

Format: As a [role], I want [capability], so that [benefit].

Each story must have acceptance criteria:
- Given [context], when [action], then [expected result]

## 4. Technical Requirements
- Performance targets (latency, throughput)
- Scalability requirements
- Availability / SLA targets
- Data storage and retention needs

## 5. Security Requirements
- Authentication and authorization model
- Data protection (encryption at rest/transit)
- Compliance requirements (OWASP, GDPR, SOC2, etc.)
- Threat model considerations (STRIDE categories)

## 6. Constraints
- Technology stack constraints
- Timeline / deadline
- Budget / resource limitations
- Integration dependencies

## 7. Success Metrics
- How will we measure success?
- Key Performance Indicators (KPIs)

## 8. References
- Links to research sources
- Related internal documents
- Industry standards cited
